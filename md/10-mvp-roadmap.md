# 10 · MVP 路线图与里程碑

## 1. 阶段划分

```
Phase 0 ─ 规划确认 ─────────────────────────► 已完成
Phase 1 ─ 脚手架 + 数据层 ─────────────────► 1–2 周
Phase 2 ─ 正文 + 识别 + 预览（核心价值）──► 2–3 周
Phase 3 ─ 角色库 + 面板 + 关系网 ──────────► 2 周
Phase 4 ─ 地图代码生成 + 设定 ─────────────► 2 周
Phase 5 ─ 体验打磨 + 二期功能 ─────────────► 2 周
```

**MVP 可发布点：** Phase 4 结束（正文识别 + 角色关系网 + 地图代码渲染 + 设定四模板）

## 4. Phase 2 · 正文与识别闭环（**最优先**）

### 任务
- [ ] 正文三栏 UI（章节列 | 编辑 | 预览 name|value 列表）
- [ ] 粘贴/保存/章节切换（章节仅存正文）
- [ ] 导入：txt / docx / 番茄格式
- [ ] LLM **两步**识别：Step1 角色名/昵称匹配 → Step2 信息提取
- [ ] 预览：上文角色库已有 + 下文可编辑 name|value 表
- [ ] **仅最新章** commit 门禁 + 删后续章工作流提示
- [ ] 同名作者裁决弹窗（Step 1 阻断）
- [ ] Commit 写入角色库 + 出场时间线 + history
- [ ] 预览仅内存，不入库
- [ ] API 设置页（文本 LLM，用户自备 Key）

### 验收标准（必须全部通过）

| # | 用例 |
|---|------|
| 1 | Step 1 识别「三哥」上下文匹配到「张三」 |
| 2 | Step 2 仅对匹配角色提取字段 |
| 3 | 预览上文=角色库已有，下文=name\|value 可编辑 |
| 4 | 非最新章「确认更新」禁用 |
| 5 | 最新章 commit 后出场时间线更新 |
| 6 | 切换章节后预览消失，正文仍在 |
| 8 | 无 API 配置时友好提示 |
| 9 | 完整字段集 + 主角关系 proximity + 同名裁决 |

### 测试
- `diffCharacter` 单元测试 ≥ 10 cases
- 识别 JSON 解析 + sanitize 测试
- 同名消歧 + 面板词条匹配测试
- 集成测试：mock LLM → preview → commit

## 8. 风险与缓解

| 风险 | 影响 | 缓解 |
|------|------|------|
| LLM 识别准确率不足 | 用户不信任预览 | 严格模式 + excerpt + 内联编辑 + 同名作者裁决 |
| LLM 生成地图代码质量不稳 | 地图不可用 | 强约束 Prompt（SVG + data-place-id）；作者可查看/编辑代码；保留上一版 |
| 关系网性能 | 大书卡顿 | 分层显示、聚合节点 |
| MVP scope 较大 | 延期 | Phase 2/3 优先；思维导图/冲突检测明确放 Phase 5 |
| 与 v2 预期落差 | 用户困惑 | 独立新项目，README 明确不提供 v2 导入 |

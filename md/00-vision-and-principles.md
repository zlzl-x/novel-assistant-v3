# 00 · 产品愿景与设计原则

## 1. 一句话定位

**帮网文作者在边写边改的过程中，自动维护「谁、在哪、什么境界、和谁什么关系」，减少吃书与设定冲突。**

不是写作软件（不替代 Word/语雀），不是 AI 代写工具，而是**设定记忆层 + 智能校对层**。

---

## 2. 目标用户画像

### 主要用户
- 连载长篇网文作者（玄幻、仙侠、都市异能等设定密集型题材）
- 习惯在外部编辑器写作，将章节**粘贴或上传**到本工具做设定维护
- 多线叙事、配角众多，容易忘记早期设定

### 次要用户
- 完结后修订的作者（批量导入全书做设定归档）
- 同人/群像作者（关系网可视化需求强）

### 非目标用户
- 需要富文本排版、发布的作者 → 应导出到专业写作平台
- 短篇、现实主义、设定极简的作品 → 工具价值有限

---

## 3. v2 为何「感觉没什么用」——教训归纳

基于 v2 代码与交互分析，推测核心问题：

| 问题 | 表现 | v3 对策 |
|------|------|---------|
| **价值对象错位** | v2 聚焦「主角面板/物品技能」，多数作者更需要「全书角色与世界观」 | 面板词条归入各角色卡片 + 角色库 + 地图 + 自定义设定 |
| **识别结果难信任** | 一步合成面板，作者看不清「改了什么」 | 严格模式双栏 diff 预览，可内联编辑，逐项确认后才写入 |
| **可视化弱** | 列表展示无法感受人物关系与地理结构 | 球状关系网 + LLM 生成代码地图（真实地理关系） |
| **工作流重** | 必须先建章节、保存、再识别 | 支持直接粘贴即识别，章节可后补 |
| **设定结构僵化** | 只有固定面板字段 | 设定页模块化 + 四内置模板（可空白） |
| **入口不清晰** | 打开即工作区，缺少全局感 | 默认进正文；地图/角色/设定分模块侧栏 |

## 4. 设计原则

### P1 · 作者主权（Author in Control）
- AI 只**提取预览**，不**擅自**修改角色库，**不提供创作建议**
- 所有识别结果必须经过预览确认（**严格模式**，无宽松自动合并）
- 预览字段可内联编辑；作者可随时手动覆盖
- 同名无法确认时，**作者绝对裁决**

### P2 · 防漏优于防错（Recall over Precision）
- 识别阶段宁可多提取候选，再在预览里由作者删减
- 地点、境界、别名、称号等字段设计为**可追加列表**，而非单值覆盖
- 提供「本章未处理的角色提及」提醒

### P3 · 边写边长（Grow as You Write）
- 允许空项目启动：只有一段地图说明也能生成初版地图
- 角色可只有名字，详情随章节逐步充实
- 关系网边距随登场角色自动扩展，不要求预先建全人设

### P4 · 最小干扰布局（Minimal Chrome）
- 正文区：章节列表极窄、编辑区居中、预览在右
- 全局左侧：角色名列表常驻但可折叠，服务于快速检索
- 不在写作流中弹大量模态框

### P5 · 模块即工具（Tools, Not Templates）
- 设定页不提供「正确的大纲格式」
- 提供块编辑器、思维导图、表格、自由文本等**工具**
- 预设模板可选，但不强制

### P6 · 本地优先（Local First）
- 作品数据默认存本地，**存储目录用户自选**（顶栏设置）
- 正文识别：Step 1 **仅发送章节正文**；Step 2 **按登场角色逐人**发送该角色 current 字段 + 本章提取（**禁止**全书角色库快照）
- 用户自备文本 LLM API Key（地图与识别共用，无需图像 API）
- 支持导出/备份整本书设定包

---

## 5. 成功指标（定性 + 定量）

### 定性
- 作者能在 30 秒内找到「某配角上次出场时的境界」
- 识别预览能让作者明确看到「相对上一版多了/改了什么」
- 关系网一眼能看出主角与核心配角的位置

### 定量（MVP 后可测）
- 单章识别 → 确认更新全流程 < 3 分钟（5000 字章节）
- 角色名搜索响应 < 200ms（500 角色规模）
- 设定冲突检测命中率（与作者自报吃书点对比）

---

## 6. 产品边界（明确不做）

- ❌ AI 续写、代写章节正文
- ❌ 发布到阅文/起点等平台
- ❌ 多人实时协同编辑（v1 不做，可预留导出）
- ❌ 替代专业思维导图软件的全部能力（做轻量嵌入即可）
- ❌ 自动判定剧情好坏或文风

---

## 7. 术语表

| 术语 | 含义 |
|------|------|
| **作品（Project）** | 一部小说，包含地图、角色、章节、设定模块 |
| **角色库（Character Registry）** | 全书角色的权威数据源 |
| **识别（Recognition）** | 从正文片段提取角色/地点/境界等结构化信息 |
| **预览（Preview）** | 识别结果与角色库 diff，未写入前的暂存态 |
| **确认更新（Commit）** | 作者批准后写入角色库，并记录变更日志 |
| **设定模块（Setting Module）** | 作者在设定页自定义的一块内容（大纲块、思维导图块等） |
| **关系网（Relation Graph）** | 以主角为中心（或群像多中心）的角色关系可视化 |
| **角色面板（Character Panel）** | 每个角色独立的词条区（功法/技能、法宝/装备等），原 v2 主角面板能力按角色拆分 |

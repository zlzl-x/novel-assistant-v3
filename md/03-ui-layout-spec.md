# 03 · 界面布局与交互规格

## 1. 正文模块（三栏核心）

### 1.1 左栏 · 章节列表（极简）

**设计目标：** 占空间最小，章号一目了然。

```
折叠态（默认）     hover / 固定展开
┌────┐            ┌──────────────┐
│ 01 │            │ 01  初入宗门  │
│ 02 │            │ 02  外门试炼  │
│ 03 │            │ 03  ···      │
│ +  │            │ +  新建章节   │
└────┘            └──────────────┘
 48px               180px
```

**交互：**
- 单击切换章节，未保存时提示
- 右键：重命名、删除、在此章后插入
- 拖拽排序（更新 `chapter.number`）
- 当前章左侧色条标识
- 有「未处理识别」或「待确认预览」的章显示小圆点

### 1.2 中栏 · 正文编辑

**输入方式：**
1. 大文本框粘贴（主路径）
2. 上传：`.txt`、`.docx`、番茄小说格式（MVP 三者均支持）
3. 可选：只读模式查看已导入正文

**顶栏工具：**
- 章节标题（可编辑）
- 字数统计
- 「智能识别」主按钮
- 保存状态指示

**编辑体验：**
- 等宽或衬线可选（用户设置）
- 不做强富文本（避免与外部写作软件重复）
- 支持查找角色名高亮（与角色库联动，可选）

### 1.3 右栏 · 识别预览（上文已有 + 下文 name|value 列表）

**识别进度：** Step 1 角色名匹配 → Step 2 信息提取（分步进度条）

**布局：**

```
┌─────────────────────────────────────────┐
│ 预览 · 第 12 章   Step1✓ Step2✓         │
│ ⚠ 仅最新章可确认更新                     │
├─────────────────────────────────────────┤
│ ▼ 张三 · 出现 14 次                     │
│                                         │
│ 【角色库已有】                           │
│ ┌──────────┬────────────────────┐      │
│ │ name     │ value              │      │
│ ├──────────┼────────────────────┤      │
│ │ 境界     │ 筑基               │      │
│ │ 所在地   │ 青云宗             │      │
│ └──────────┴────────────────────┘      │
│                                         │
│ 【本章识别】可编辑                       │
│ ┌──────────┬────────────────────┬───┐ │
│ │ name     │ value              │ ☑ │ │
│ ├──────────┼────────────────────┼───┤ │
│ │ 境界     │ 金丹               │ ☑ │ │
│ │ 所在地   │ 万妖岭             │ ☑ │ │
│ └──────────┴────────────────────┴───┘ │
├─────────────────────────────────────────┤
│            [ 确认更新角色库 ]            │
└─────────────────────────────────────────┘
```

**交互规则：**
- **上文：** 从角色库拉取的 current 字段，name|value 只读表
- **下文：** 本章识别结果，name|value **可编辑**，每行 checkbox
- 变更行高亮；支持手动增删行
- **严格模式：** 须勾选 + 确认
- **最新章门禁：** 非最新章禁用「确认更新」，提示删后续章重贴
- **同名消歧：** Step 1 歧义须先裁决才能进入 Step 2
- **预览不入库：** 切换章节后预览消失，仅正文保留

**空状态：**
- 未识别：「粘贴正文后点击智能识别」
- 识别中：「Step 1 识别角色名…」→「Step 2 提取信息…」

## 2. 角色模块

### 2.1 球状关系网

**视觉规则：**
- **默认模式：** 单主角固定中心（金色/最大节点）
- **群像模式：** 作者可切换多中心布局（多个核心节点）
- 外层节点距离/大小反映与主角**关系远近量化值**（识别产出 + 可手动调整）
- 边：关系类型（师徒、仇敌、同门、亲属…）+ 强度

**交互：**
- 滚轮缩放、拖拽平移
- 点击节点 → 右侧或底部抽屉展示人物卡片 + **角色面板词条**
- 双击 → 全屏详情页
- 从左侧角色列表点击 → 相机平滑移动到节点

**技术候选：** D3 force radial layout / Cytoscape.js / vis-network

### 2.2 人物卡片 + 角色面板

防吃书优先，分两层展示：

**基础卡片字段：**

| 字段 | 说明 |
|------|------|
| 姓名 / 别名 | 列表；同名须带身份标识 |
| 身份 / 称号 | 当前最新，历史可展开 |
| 境界 / 等级 | 自由文本 |
| 所在地 | 文本（不自动关联地图） |
| 所属势力 | 文本 |
| 与主角关系 | 类型 + 远近量化值（1–5） |
| 最近出场 | 章节号 + 出场时间线条目 |
| 备注 | 作者自由文本 |

**角色面板词条区（原 v2 主角面板能力，按角色独立）：**

| 词条类型 | 说明 |
|----------|------|
| 功法/技能 | 列表，识别可提取 |
| 法宝/装备 | 列表，识别可提取 |
| 年龄/寿命 | 文本或数值 |
| 自定义词条 | 作者预定义面板模板后，识别自动匹配词条名 |

- 面板词条**仅**通过识别预览确认或手动编辑更新，**不**在识别后静默自动更改
- 系统**只预览、不建议**，不提供 AI 创作改写

**出场时间线（详情页）：**

| 章节 | 标题 | 出现次数 |
|------|------|----------|
| 01 | 初入宗门 | 3 |
| 12 | 万妖岭 | 14 |

- 字段时间线：各字段 `history` 按章节展开（境界、所在地等）

## 3. 地图模块

### 3.1 布局结构

```
┌──────────────┬─────────────────────────────────────────────┐
│ 世界选择 ▾   │                                             │
│ 🔍 搜索地点   │   ┌─────────────────────────────────────┐   │
│              │   │  [沙箱 iframe：LLM 生成的交互地图]    │   │
│ 🌍 东洲      │   │   青云宗 ── 天元城                    │   │
│  ├ 青云宗    │   │      ╲                                │   │
│  └ 万妖岭    │   │       ╲  西荒                        │   │
│              │   └─────────────────────────────────────┘   │
│ [编辑提示词] │   hover → 区域突出 + 地名浮层               │
│ [生成地图]   │   [重新生成] [查看生成的代码]               │
└──────────────┴─────────────────────────────────────────────┘
```

### 3.2 视图模式
1. **代码渲染地图（MVP 主视觉）：** 文本 LLM 生成 HTML/SVG，沙箱 iframe 渲染，表达真实地理关系
2. **树状层级（MVP 辅助）：** 世界 → 区域 → 地点，左侧导航，与地图双向联动
3. **增量局部更新（第二期）：** 增删地点后补丁更新代码，不必整图重生成

### 3.3 与正文识别的关系
- **不联动：** 识别章节不自动更新地图
- 地图由作者在地图页通过**提示词**调用**同一文本 LLM** 生成代码
- **不需要**另接图像 API

## 4. 设定模块

### 4.1 页面结构

```
┌────────────────────────────────────────────┐
│ [+ 添加模块 ▾]  富文本 | 思维导图 | 表格 … │
├────────────────────────────────────────────┤
│ ┌─ 模块：全书大纲 ─────────────── [⋮] ─┐ │
│ │  （富文本编辑区）                      │ │
│ └────────────────────────────────────────┘ │
│ ┌─ 模块：力量体系 ─────────────────────┐ │
│ │  （表格）                              │ │
│ └────────────────────────────────────────┘ │
└────────────────────────────────────────────┘
```

### 4.2 模块类型（分期）

| 类型 | MVP | 说明 |
|------|-----|------|
| 富文本块 | ✓ | 标题 + Markdown/轻量 WYSIWYG |
| 清单块 | ✓ | 勾选列表 |
| 表格块 | ✓ | 自定义列 |
| 内置四模板 | ✓ | 大纲 / 力量体系 / 人设表 / 伏笔追踪（可空白插入） |
| 思维导图 | 二期 | 嵌入轻量组件 |
| 关联块 | 二期 | 嵌入某角色/地图节点的只读摘要 |

## 6. 响应式与窗口尺寸

| 宽度 | 行为 |
|------|------|
| ≥ 1440px | 完整三栏 + 双全局侧栏 |
| 1280–1439 | 预览栏可默认折叠，按钮展开 |
| < 1280 | 角色名侧栏自动折叠；预览改为底部抽屉 |

**最小支持宽度：** 1024px（桌面应用）

---

## 7. 无障碍与可读性

- 对比度 WCAG AA
- 关系网节点除颜色外有形状区分
- diff 预览：红/绿之外用图标（+ / ~ / −）
- 支持系统字体缩放

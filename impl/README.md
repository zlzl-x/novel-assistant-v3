# 计划实施（Implementation）

本目录是 **可逐步执行的开发实施说明**，与 [`md/`](../md/)（产品计划说明）分离：

| 目录 | 用途 | 是否随开发修改 |
|------|------|----------------|
| [`md/`](../md/) | 产品愿景、功能规格、数据模型、开放问题决策 | 仅产品变更时改 |
| **`impl/`** | 分步实施任务、文件清单、难点与解法、验收命令 | 随开发进度勾选更新 |

## 如何使用

1. **按 Step 编号顺序执行**（Step 01 → 15），不要跳步；每步「前置依赖」不满足则先完成依赖步。
2. 每步文档结构统一：
   - 目标与产出
   - 前置依赖
   - 详细任务清单（可勾选）
   - 建议文件/目录
   - **难点与解决方案**
   - 验收标准与测试命令
   - 参考计划文档链接
3. 完成一步后，在本文件底部「进度表」打勾，再进入下一步。

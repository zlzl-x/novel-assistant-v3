# 小说创作助手 v3

面向长篇网文作者的**设定记忆工具**：在外部编辑器写作，将章节粘贴到本软件，自动提取角色、境界、关系与地点变化，减少吃书与设定冲突。

- 不是写作软件，不替代 Word / 语雀
- 不是 AI 代写工具，只提取预览、由作者确认后写入
- 数据本地存储，LLM API Key 由用户自行配置

---

## 安装与运行

### 普通用户：下载安装包

1. 打开 [GitHub Releases](https://github.com/zlzl-x/novel-assistant-v3/releases)
2. 下载最新版 `novel-assistant-v3-setup-x.x.x.exe` 并安装
3. 首次启动 → **设置** 中填写 LLM API Key → 顶栏选择 **存储目录**

### 开发者：从源码运行

**环境：** Node.js ≥ 20、pnpm 9

```bash
git clone https://github.com/zlzl-x/novel-assistant-v3.git
cd novel-assistant-v3
pnpm install
pnpm dev
```

### 自行打包

```bash
pnpm install
pnpm --filter @novel-assistant/desktop build:installer
```

产物：`apps/desktop/dist/小说创作助手 v3 Setup 0.0.1.exe`

## 程序操作说明

### 首次使用

| 步骤 | 操作 |
|------|------|
| 1 | 顶栏点击 **存储目录**，选择作品数据保存位置（所有作品共用此目录） |
| 2 | 进入 **设置**，选择 API 服务商预设或自定义 Base URL，填写 API Key 并保存 |
| 3 | 首页 **新建作品**，输入书名后进入工作台 |

> API Key 仅加密保存在本机，不会上传到 GitHub 或任何远程服务器。

### 界面布局

```
顶栏：存储目录 | 作品名 | 设置 | 导出
├─ 最左栏：全书角色名列表 + 搜索（各页常驻，可折叠）
├─ 侧边导航：正文 | 地图 | 角色 | 设定
└─ 主内容区：当前页面
```

### 正文（核心工作流）

1. **导入章节**
   - 在中间编辑区粘贴正文，或点击导入（支持 `.txt`、`.docx`、番茄格式）
   - 左侧章节栏可新建、切换、排序、删除章节

2. **智能识别**
   - 点击 **智能识别**，系统分两步调用 LLM：
     - **Step 1**：仅发送章节正文，提取登场角色名与初步字段
     - **Step 2**：按角色逐个发送该角色现有资料 + 本章提取结果，生成更新建议
   - 若出现同名歧义，会弹出裁决框，由作者指定对应角色

3. **预览与确认**
   - 右侧预览面板显示「当前库中值 → 建议值」对比
   - 可逐行勾选要采纳的字段，也可内联编辑
   - 点击 **确认更新** 写入角色库
   - **仅最新章节允许确认**；编辑历史章节时需先删除其后章节

4. **角色库联动**
   - 确认后自动更新角色境界、别名、关系、出场记录等
   - 左侧角色列表与角色页关系网同步刷新

### 地图

1. 左侧选择或新建 **世界/地图**
2. 填写 **地图描述提示词**（如地形、城池、山脉、河流、关键地点）
3. 可手动添加地点节点作为补充上下文
4. 点击 **生成地图**（需已配置 API Key，通常需数分钟）
5. 主区域沙箱渲染 LLM 生成的 HTML/SVG 地图
   - 区域可点击，悬停高亮
   - 与正文识别独立，不会自动从章节更新地图

### 角色

- **关系网**：球状布局，默认以主角为中心；群像作品可切换多中心
- **点击节点**：打开人物卡片，查看境界、别名、关系、角色面板词条
- **搜索**：左侧栏支持模糊搜索角色名与别名，显示境界与最近出场章
- **合并角色**：可将误分的重复角色合并

### 设定

1. 点击 **添加模块**，选择工具类型：
   - 富文本、清单、表格（内置模板：大纲 / 力量体系 / 人设表 / 伏笔追踪）
2. 各模块独立编辑，内容保存在本地数据库
3. 设定页与正文识别互不干扰，需作者手动维护

### 导出

顶栏 **导出** 可将当前作品数据打包备份，便于迁移或归档。

## 项目功能

| 模块 | 功能 | 说明 |
|------|------|------|
| **作品管理** | 新建 / 打开 / 删除作品 | 数据存于用户自选目录 |
| **章节管理** | CRUD、排序、导入 | 支持 txt / docx / 番茄格式 |
| **正文识别** | 两步 LLM 提取 + 预览确认 | 严格模式，作者主权；仅最新章可 commit |
| **角色库** | 角色卡片、别名、境界、出场记录 | 识别结果经预览后写入 |
| **关系网** | vis-network 球状图 | 单主角 / 多中心，节点点击查看详情 |
| **地图** | LLM 生成 HTML 地图 + 沙箱渲染 | 提示词驱动，可交互区域 |
| **设定模块** | 自定义块编辑器 | 富文本 / 清单 / 表格 + 四内置模板 |
| **设置** | API Key、模型、Base URL | 本地加密存储 |
| **导入导出** | 章节导入、作品导出 | 本地文件操作 |

### 设计要点

- **作者主权**：AI 只建议，不擅自改库；所有写入经预览确认
- **防漏优于防错**：宁可多提取候选，作者在预览中删减
- **边写边长**：允许空项目启动，角色可以只有名字
- **本地优先**：默认离线存储，识别时仅向 LLM 发送必要文本

## 项目结构

```
novel-assistant-v3/
├── apps/
│   └── desktop/                 # Electron 桌面应用
│       ├── src/
│       │   ├── main/            # 主进程：IPC、LLM 客户端、导入导出、存储路径
│       │   ├── preload/         # 预加载脚本，暴露安全 API 给渲染进程
│       │   └── renderer/        # Vue 3 渲染进程
│       │       └── src/
│       │           ├── views/           # 页面：正文、地图、角色、设定、设置
│       │           ├── components/      # UI 组件（布局、正文、角色、地图、设定）
│       │           ├── stores/          # Pinia 状态（章节、角色、地图、预览等）
│       │           ├── services/        # 识别、地图生成、LLM 调用封装
│       │           └── router/          # 路由
│       ├── electron-builder.yml # 打包配置
│       └── package.json
│
├── packages/
│   ├── core/                    # 核心业务逻辑（无 UI 依赖）
│   │   └── src/
│   │       ├── recognition/     # 两步识别流水线、提示词、预览构建
│   │       ├── commit/          # 预览确认后的字段写入逻辑
│   │       ├── map/             # 地图 LLM 生成、HTML 校验与消毒
│   │       ├── graph/           # 关系网数据转换
│   │       ├── character/       # 角色合并等
│   │       ├── import/          # 番茄 / 章节拆分解析
│   │       ├── settings/      # 设定模块模板与搜索
│   │       └── models/          # 领域模型与 IPC 类型
│   │
│   └── db/                      # SQLite 数据层
│       └── src/
│           ├── repositories/    # 作品、章节、角色、地图、设定等仓储
│           ├── migrations/      # 数据库迁移
│           └── database-service.ts
│
├── scripts/                     # 辅助脚本
│
├── md/                          # 产品规格文档（详细设计）
├── impl/                        # 分步实施手册（开发用）
├── package.json                 # monorepo 根配置
└── pnpm-workspace.yaml
```

### 技术栈

| 层级 | 技术 |
|------|------|
| 桌面壳 | Electron 33 + electron-vite |
| 前端 | Vue 3 + Pinia + Vue Router + Naive UI + Tailwind |
| 主进程 | TypeScript，IPC 桥接 |
| 业务逻辑 | `@novel-assistant/core`（纯 TS，可单测） |
| 数据库 | SQLite（better-sqlite3）via `@novel-assistant/db` |
| 关系网 | vis-network |
| 校验 | Zod |
| 包管理 | pnpm workspace monorepo |
